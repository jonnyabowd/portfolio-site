HUEY Weather

Challenge

Merging a passion for weather forecasting, minimalist design, and trend analysis, I set out to create a web-based 168hr weather data visualization of temperature and chance of precipitation that is 100% non-alphanumeric. The goal is to condense a large and complicated amount of forecast data into a simplistic, high-level view that can be quickly interpreted at a glance.

Technology Used

  • HTML
  • CSS
  • Vanilla Javascript
  • Darksky API
  • Google Places API
  • Github

Solution

Using a simple color matrix charting temperature and chance of precipitation along the respective X and Y axes, create a week long cell grid divided by hour and plot the appropriate color values in the appropriate color cells and indicate which cell corresponds to the current time.

High-Level Code Logic

  1. User enters location
  2. Use Google Places API to determine X,Y coordinates
  3. Fetch data from DarkSky using X, Y coordinates and return extended hourly forecast for the next 168 hours (1 week)
  4. Create a 7 x 24 space grid (168 total spaces) within a GUI representing the time (in hours) and days
  5. Starting with Sunday at 12am in the top left, use temperature and precipitation for each hour to style the background color on each space of the grid
  6. Indicate which hour in which day is the current hour
  7. Set timer to rerun application at the top of each hour to update data as long as the browser window is open

Homepage Design

A single location input and data toggle control are all that powers this experience.

Simple and beautiful, with a wealth of data to be interpreted at a glance. HUEY enables you to see trends in the weather around you.
